# Tool Site Project Brief

## Project Goal

Create a long-term SEO-focused tool website using Codex as the development assistant.

The site should be treated as a real web asset, not a quick AI-generated demo. The goal is to build useful tools that can attract Google search traffic over time, then monetize through Google AdSense and relevant affiliate links.

## Background

We compared two ways to build the site:

1. AI website builders such as Hostinger Horizons.
2. Codex working directly in a real code project.

The conclusion is to use Codex for this project.

Hostinger Horizons is better for fast prototypes and simple no-code publishing. Codex is better for a serious tool site because it can work with real source code, Git, tests, deployment, SEO files, and future custom logic.

## Why Codex Is Preferred

Codex is closer to an AI programmer than an AI website generator.

Advantages:

- Full control over source code.
- Easier to add real tool logic instead of relying on generated templates.
- Better for SEO page structure, performance, and future iteration.
- Easier to deploy to Netlify, Vercel, Cloudflare Pages, or another host.
- Less platform lock-in.
- Better long-term fit if the site becomes a real asset.

The current plan is to use Codex together with GitHub and a free or low-cost hosting platform such as Netlify or Vercel.

## Business Direction

Build a small niche tool site, not a random collection of unrelated tools.

A focused tool cluster is better for SEO than many unrelated tools because Google can understand the site topic more clearly.

Possible niches:

- English word tools.
- E-commerce calculators.
- Shopify seller tools.
- YouTube creator tools.
- AI prompt tools.
- PDF or text utilities.

The first site should avoid extremely competitive keywords like "calculator" or "word unscrambler" unless there is a more specific long-tail angle.

## Recommended MVP

Start with one focused niche and build 3-5 related tools.

Example: Shopify and e-commerce calculator site.

Possible first tools:

- Shopify profit calculator.
- PayPal fee calculator.
- Stripe fee calculator.
- Product pricing calculator.
- ROAS calculator.

This niche has strong commercial intent and natural affiliate opportunities, such as Shopify, SEO tools, payment tools, or business services.

## MVP Pages

The first version should include:

- Home page.
- One primary tool page.
- 2-4 related tool pages.
- About page.
- Contact page.
- Privacy Policy.
- Terms of Use.
- Disclaimer or Affiliate Disclosure.

Each tool page should include:

- Clear H1.
- Short explanation.
- The actual interactive tool.
- Helpful SEO content below the tool.
- FAQ section.
- Internal links to related tools.
- Reserved ad placement areas for future AdSense.

## SEO Principles

The site should be built for useful search traffic, not mass-produced AI pages.

Important principles:

- Pick keywords with clear intent.
- Check Google search results before building a tool.
- Avoid topics where page one is dominated only by major sites.
- Prefer long-tail tool keywords.
- Build a related group of tools in one niche.
- Make the tool genuinely useful before adding ads.
- Keep pages fast and mobile-friendly.
- Use original text and avoid copying competitors.

## Monetization

Primary monetization options:

- Google AdSense after the site has enough useful content.
- Affiliate links where they naturally fit user intent.
- Later: sponsored placements or lead generation if the site gets traffic.

Do not apply for AdSense too early if the site is just an empty tool page. The site should first have working tools, useful explanation pages, privacy policy, terms, contact page, and enough original content.

## Technical Direction

Recommended stack:

- Next.js or a simple React/Vite app.
- TypeScript if the project starts cleanly with it.
- Static deployment first.
- No database for the first version unless a tool truly needs one.
- GitHub repository for version control.
- Netlify, Vercel, or Cloudflare Pages for deployment.

Keep the first version simple. Avoid accounts, dashboards, payments, complex backend logic, or user-generated content until there is proof of demand.

## First Build Criteria

The first public version is successful if:

- The site loads quickly on mobile and desktop.
- At least one primary tool works correctly.
- Related tool pages exist and are internally linked.
- Basic SEO metadata is present.
- Legal/basic trust pages exist.
- The project can be deployed without manual hacks.
- The code is easy for Codex to continue editing.
